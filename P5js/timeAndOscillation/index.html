My object based programming and thinking-in-parameters helped a lot here! This will also make the code modular and I can “tune” parameters to result in very different visual results.

Cool! So I also wanted to explore how I can control amplitude - initially this was simple, having a variable called "amplitude" that is multipled directly outside of sin() for my y-movement:

Seen above, example of varying the amount of waves drawn AND changing the amplitude by some constant value in pixel height!

Here's where things got interesting - constricting amplitude with a tapering effect, in this case, constrict to a sphere radius r. In my amplitude calculation " r * sqrt(1- pow((x/r), 2))" modulates the vertical displacement of the wave at each horizontal position x, resulting in a wave shape that follows a circular arc.

I overlayed an ellipse to confirm cicular restriction and it starts looking really cool at increasing number of waves!
